# Take-Home Recipe

Recommended mode: `controlled or council`

## Use When

The output will be evaluated by an interviewer or external reviewer.

## Flow

1. Clarify problem, constraints, and scoring criteria.
2. Create a short plan with visible tradeoffs.
3. Build in small deliverable slices.
4. Verify, review, proof-pack, and polish README/run instructions.

## Done When

The project is easy to run, easy to inspect, and honest about limitations.
