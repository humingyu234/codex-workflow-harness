# Open-Source PR Recipe

Recommended mode: `controlled`

## Use When

The change will be sent to another repository's maintainers.

## Flow

1. Keep the scope small and maintainer-friendly.
2. Match existing code and test style.
3. Add focused tests for the changed behavior.
4. Prepare a reviewable diff and concise PR explanation.

## Done When

The specific issue fixed, tests run, and low-risk scope are clear.
